# Plan 02-03: HTTPS SNI Extraction and Filtering

## Objective

Wire blocklist filtering into the HTTPS CONNECT tunnel path. Modify `connect_tunnel.py` so that `tunnel_connect()` checks both the CONNECT target host AND the TLS SNI against the blocklist. Blocked hosts get 403 (CONNECT stage) or silent close (SNI stage). Allowed hosts get normal tunnel with ClientHello forwarded to target before relay.

**Purpose:** HTTPS domain filtering via SNI is essential — HTTPS traffic is encrypted, so filtering must happen at the TLS handshake layer before the tunnel is established.

**Output:** Modified `src/connect_tunnel.py` with two-stage filtering (CONNECT host + SNI).

    1. **Extend the import line** (already added in Task 1) to also import `extract_sni`:
       ```python
       from src.filter import blocklist, BLOCK_PAGE, extract_sni
       ```

    2. **After the `200 Connection Established` response** (after `wfile.write(b"HTTP/1.1 200 Connection Established\r\n\r\n")` and `wfile.flush()`), add ClientHello read + SNI extraction + blocklist check:

       ```python
       # Phase 2: Read ClientHello and check SNI
       clienthello_bytes = None
       try:
           client_sock.settimeout(5.0)
           clienthello_bytes = client_sock.recv(4096)
       except (socket.timeout, OSError):
           pass  # No ClientHello — allow through (CONNECT host already cleared)

       if clienthello_bytes:
           sni = extract_sni(clienthello_bytes)
           if sni and blocklist.is_blocked(sni):
               # Silently close — client sees connection reset
               target_sock.close()
               return
           # Forward ClientHello to target BEFORE relay loop
           target_sock.sendall(clienthello_bytes)
       ```

    3. **Placement**: This code goes between `wfile.flush()` and `client_sock.setblocking(False)`:
       - Right after the `200 Connection Established` response block
       - Right before the `# 3. Configure sockets for non-blocking I/O` section
       - The `settimeout` is temporary; `setblocking(False)` resets the socket for `select()` after SNI check

    4. **Critical design behaviors** (from RESEARCH.md Pitfalls):
       - **Pitfall 3 (ClientHello re-injection)**: After extracting SNI, the ClientHello bytes are stored in `clienthello_bytes`. If allowed, `target_sock.sendall(clienthello_bytes)` sends them to the target BEFORE entering the relay loop. Without this, the target waits for a ClientHello that the proxy already consumed.
       - **Pitfall 4 (CONNECT host vs SNI mismatch)**: If SNI extraction fails (timeout or no SNI extension) or returns None, the connection is still allowed through (CONNECT host check already passed). This handles IP-based CONNECT targets that carry a different SNI.
       - **Pitfall 1 (ClientHello not yet available)**: The 5-second timeout on `client_sock.recv()` handles the race condition where the 200 response hasn't been processed by the client yet. If timeout occurs, `clienthello_bytes` stays None and the connection proceeds normally.
       - **Pitfall 2 (SNI not present)**: If `extract_sni()` returns None (no server_name extension), the connection is allowed. The CONNECT host check already blocked trivial bypasses.

    5. **Preserve all existing relay code**: After the SNI check block, the existing socket configuration and `select()` relay loop remain unchanged. The relay loop (`select.select()`) is entered only after SNI processing is complete.

    6. **The `extract_sni` import is already present** from extending the import line in step 1. `socket` module is already imported at the top of the file.
