    Crear src/http_relay.py con la función `forward_request(client_sock, wfile, request_line, headers, body)`.

    **Firma:** `def forward_request(client_sock: socket.socket, wfile: io.BufferedWriter, request_line: str, headers: dict, body: bytes) -> None`

    **Lógica:**

    1. **Parsear request line:**
       - Split en partes: `method, full_url, http_version = request_line.split()`
       - Usar `urllib.parse.urlparse(full_url)` para extraer hostname, port, path, query
       - Determinar puerto: `parsed.port or 80`
       - Construir path: `parsed.path or '/'`, si hay query: `path + '?' + parsed.query`
       - **Validar que parsed.hostname no sea None** (Pitfall 5). Si lo es, escribir `HTTP/1.1 400 Bad Request\r\n\r\n` y return.

    2. **Validar Content-Length para body (Pitfall 1):**
       - Si `body` tiene datos pero `len(body) < content_length`, hacer loop recv hasta recibir todos los bytes.
       - Usar `BUFFER_SIZE` (de config) para el tamaño del recv.
       - Usar `client_sock.settimeout(CONNECT_TIMEOUT)` para el timeout.

    3. **Conectar al servidor destino:**
       - Crear socket TCP: `target_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)`
       - Setear timeout: `target_sock.settimeout(CONNECT_TIMEOUT)`
       - Conectar: `target_sock.connect((host, port))`
       - Si connection fails (socket.gaierror, ConnectionRefusedError, timeout): escribir `HTTP/1.1 502 Bad Gateway\r\nContent-Length: 0\r\n\r\n` al wfile, cerrar target_sock, return.

    4. **Reconstruir request en origin-form (path-only):**
       - `new_request_line = f"{method} {path} {http_version}\r\n"`
       - Headers a reenviar: iterar `headers.items()`, incluir todos EXCEPTO:
         - `proxy-connection`, `proxy-authorization` (headers hop-by-hop propios de proxy)
       - Host header: reescribir como `f"Host: {host}:{port}"` (usar el host parseado, no el del header original — Pitfall 5)
       - Connection header: agregar `Connection: close` para evitar keep-alive (por simplicidad, Phase 1)
       - Armar string de headers: `"Key: Value\r\n"` para cada header
       - Terminar con `"\r\n"` (doble CRLF)

    5. **Enviar request al destino:**
       - `target_sock.sendall(new_request_line.encode('utf-8') + headers_str.encode('utf-8') + body)`

    6. **Relayar respuesta al cliente:**
       - Loop: `data = target_sock.recv(BUFFER_SIZE)` hasta que data esté vacío (EOF)
       - Cada chunk: `wfile.write(data); wfile.flush()`
       - Manejar `ConnectionResetError`, `BrokenPipeError`, `OSError` — salir del loop sin crash

    7. **Cleanup (Pitfall 3 — connection leaks):**
       - `try/finally` alrededor de la conexión al destino
       - En finally: `target_sock.close()`
       - NO cerrar `client_sock` — eso lo maneja StreamRequestHandler automáticamente

    **Importante:** No usar `http.client.HTTPConnection` — usar raw socket + manual reconstruction. Esto da control total sobre headers y evita comportamientos inesperados del cliente HTTP.

    **Incluir en el módulo:**
    - Docstring: "HTTP request forwarding — parses proxy requests, forwards to target server, relays response."
    - Imports necesarios: `socket`, `urllib.parse`, `from src.config import PROXY_HOST, PROXY_PORT, BUFFER_SIZE, CONNECT_TIMEOUT`
    - NO incluir bloque `if __name__ == "__main__"` — es un módulo de función, no script.
