  <action>
    Crear src/handler.py con la clase `ProxyRequestHandler(socketserver.StreamRequestHandler)`:

    1. **Método `handle(self)`:**
       - Lee request line: `self.rfile.readline()` → decodifica UTF-8 con errors='replace' → strip
       - Si request line vacío, return (cierra conexión)
       - Extrae método: `request_line.split(' ')[0].upper()`
       - Lee headers en loop:
         - `self.rfile.readline()` hasta `b'\r\n'` o línea vacía
         - Decodifica, split en ':' para key/value, lower case keys
         - Almacena en dict `self.headers`
       - Lee body si existe (solo para non-CONNECT):
         - Content-Length: `int(self.headers.get('content-length', '0'))`
         - Si > 0: `body = self.rfile.read(cl)`
       - Dispatch:
         - Si método == 'CONNECT': llama `self.handle_connect(request_line)`
         - Sino: llama `self.handle_http(request_line, dict(self.headers), body)`

    2. **Método `handle_http(self, request_line, headers, body)`:**
       - Lazy import: `from http_relay import forward_request`
       - Llama: `forward_request(self.request, self.wfile, request_line, headers, body)`
       - **Sin try/except** — el ImportError natural avisa que http_relay no existe aún (esperado hasta Plan 01-02)

    3. **Método `handle_connect(self, request_line)`:**
       - Lazy import: `from connect_tunnel import tunnel_connect`
       - Llama: `tunnel_connect(self.request, self.wfile, request_line)`
       - **Sin try/except** — el ImportError natural avisa que connect_tunnel no existe aún (esperado hasta Plan 01-03)

    4. **Atributo de clase (opcional pero recomendado):**
       - `MAX_HEADER_SIZE = MAX_HEADER_SIZE` importado de config (para referencia)

    **Diseño clave:** Los imports dentro de handle_http/handle_connect son lazy (locales al método). Esto permite que:
    - Plan 01-01 funcione: handler compila, dispatch funciona, pero cualquier request genera ImportError en http_relay/connect_tunnel (esperado)
    - Plan 01-02: http_relay.py existe → handle_http funciona. CONNECT aún falla (esperado)
    - Plan 01-03: connect_tunnel.py existe → handle_connect funciona. Ambos métodos operativos.

    Usar `self.request` para el socket raw y `self.wfile` para escribir respuestas HTTP. StreamRequestHandler provee ambos automáticamente.

    Incluir docstring de módulo: "HTTP proxy request handler — parses requests and dispatches to specialized handlers."
