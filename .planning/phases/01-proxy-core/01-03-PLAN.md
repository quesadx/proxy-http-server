    1. **Parsear target del request line:**

    2. **Conectar al servidor destino:**
       - Crear socket: `target_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)`
       - Setear timeout: `target_sock.settimeout(CONNECT_TIMEOUT)` (de config)
       - `try/except` para connect:
         - Ă‰xito: escribir `b"HTTP/1.1 200 Connection Established\r\n\r\n"` al wfile, hacer flush
         - `socket.gaierror`, `ConnectionRefusedError`, `OSError`, `socket.timeout`:
           - Escribir `b"HTTP/1.1 502 Bad Gateway\r\n\r\n"` al wfile, flush
           - Cerrar target_sock, return

    3. **Configurar sockets para non-blocking:**
       - `client_sock.setblocking(False)` â€” necesario para select.select()
       - `target_sock.setblocking(False)`

    4. **Bucle bidireccional con select.select() (Pitfall 2 â€” evitar deadlock):**
       - `sockets = [client_sock, target_sock]`
       - Loop while True:
         - `readable, _, exceptional = select.select(sockets, [], sockets, 30.0)`
         - Si exceptional â†’ break (error en socket)
         - Si not readable â†’ continue (timeout, keep polling)
         - Para cada sock en readable:
           - `other = target_sock if sock is client_sock else client_sock`
           - `try: data = sock.recv(4096)`
             - Si `not data`: return (conexiĂ³n cerrada por el peer)
             - `other.sendall(data)`
           - Except `(ConnectionResetError, BrokenPipeError, OSError)`: return

    5. **Cleanup (Pitfall 3):**
       - `try/finally` alrededor del bucle principal
       - En finally: cerrar target_sock (client_sock lo gestiona StreamRequestHandler)
       - **No cerrar client_sock** â€” el handler base lo cierra al retornar handle()
       - Manejar doble-close: cada close envuelto en `try/except OSError: pass`

    6. **RecomendaciĂ³n de buffer size:**
       - Usar 4096 para recv (tamaĂ±o comĂºn para relay â€” evita fragmentaciĂ³n excesiva)

    **Validaciones adicionales:**

    - Si target_sock se cierra (EOF), relayear los Ăºltimos datos y salir.
